<시험 이야기>
- 시험은 필기 시험
- 우리가 이론에 대한 부분은 언급하지 않고 만드는 것이 중요하다
- 전부 코드를 짜는 것이 중요하다
- 문제 주어지고 만드는 것 1개, 괄호 넣기, SQL 2~3개해서 총합 30점이다
- <html xmlns="http://www.~~"> 이런 부분은 하지 않는다, class 파일 어케 만드는지, 어케 받아와서 넘기는 지 등만 알면 된다
- 코드가 아주 복잡하지는 않음

- 처음부터 끝까지 코드를 작성하는 부분은 성적처리 프로그램 하나, 성적 처리 프로그램
class를 만들고, 소스코드를 만들어봐라 -> 제대로 된 문제는 1개

- SQL문 1~2개 작성 -> 어렵지 않은 문제, 코드 작성 못하는 사람들의 기본 점수를 위해서 내는 문제

- 나머지는 괄호 넣기 -> 큰 카테고리 7개쯤?
  - 면적 구하기
  - 리스트 추가하기 - DropDownList 등
  - 파일에 대한 부분 - 읽기 (커피 머신 예제 참조)
  - 공통 코드 부분 - commandName, commandArgument
  - 상태관리 - PreRednder
  - 유효성 검사

그룹이 7개 정도인데 괄호 넣기로 4개쯤 나온다
