<ASP.NET의 이해>
서버 - 자료나 자원을 공유
클라이언트 - 자료나 자원을 요구

웹 앱플리케이션(웹 응용 프로그램)
웹 프로그래밍 = 웹 애플리케이션 개발
서버 - 자료나 자원을 제공(IIS, Apache, Tomcat, PHP 등)
클라이언트 - 자료나 자원을 요구(웹 브라우저)
Request & Response
Backend & Frontend

웹을 통한 상호 커뮤니케이션 형태 진화 -> 다양하고 창의적 웹 애플리케이션 개발 필요성 대두

정적 웹 페이지(변경이 되지 않는 페이지들...) vs 동적 웹 페이지(새롭게 데이터를 받아오는 페이지 들...)

웹의 구성 - URL, HTTP, HTML

정적 웹 페이지 - 서버에 있는 문서가 수정되지 않는 한 항상 같은 웹 문서 응답
웹 사이트 - 웹 사이트는 정적 웹 페이지로만 구성되지 않는다?

동적 웹 페이지
클라이언트 -> 1요청 -> 웹 서버 -> 웹 애플리케이션 -> DB -> 웹 애플리케이션 -> 웹 서버 -> 2응답  -> 클라이언트

ASP.NET?
웹 애플리케이션 개발 언어 중 하나 - ASP, ASP.NET, JSP, PHP, Ruby(on Rails) 등
MS사의 ASP 차세대 버전이다

LINQ - SQL문 담고 데이터 가져오기
MVC통합 - 최종적 홈페이지 만드는데, model view controller

이벤트 기반 프로그래밍 모델 - class를 만들고 import 시킬 수 있음, 코드의 길이가 짧음
컨트롤 기반 페이지 - 부품을 조립하여 완성품을 만들 듯 쉽고 직관적으로 프로그래밍
언어 독립적인 플랫폼 제공 - 스크립트 언어뿐 아니라 커파일된 언어도 제공, 어떠한 .NET언어로도 ASP.NET 페이지 작성 가능, 만들어진 웹 페이지는 CLR이 지원되는 운영체제에서 구동 가능

열 때 VB로 구성되어져 있는 것으로 열면 VB로 코딩을 해야함... C#으로 열어야 함

Code-behind 모델
ASP의 스파게티 코드(UI, Logic 섞임)의 문제점 극복하기 위해 UI와 업무 로직 분리
사용자 UI - HTML 부분으로 사용자들에게 보여지는 부분(.aspx)
업무 로직 - 업무 처리를 위한 코드 부분(.cs)
